# Code of Conduct

## Our Standard

MindMap Care contributors should help maintain a respectful, serious, and constructive project environment.

This repository concerns health workflows, vulnerable populations, clinical trust, and public health systems. Discussion should reflect that responsibility.

Expected behavior:

- use welcoming and respectful language
- assume good intent while being precise about risks
- give and receive feedback constructively
- keep clinical, ethical, and data-protection concerns visible
- respect differences in geography, discipline, experience, language, and institutional context
- focus disagreement on ideas, evidence, and implementation details

Unacceptable behavior:

- harassment, threats, personal attacks, or discriminatory language
- dismissive treatment of clinical safety, privacy, or ethics concerns
- sharing real patient information or private operational data
- presenting MVP outputs as validated diagnoses
- knowingly misleading funders, partners, clinicians, or users about product readiness

## Health Data Responsibility

Never post real patient data, identifiable clinical details, private datasets, credentials, or production records in issues, pull requests, discussions, or commits.

If sensitive information is accidentally shared, notify the project maintainers immediately so it can be removed and remediated.

## Enforcement

Maintainers may remove content, close discussions, reject contributions, or restrict participation when behavior conflicts with this code.

For now, report concerns by opening a private communication channel with the repository owner or project maintainer. A dedicated security and governance contact should be added before any real pilot.

## Scope

This code applies to project spaces including GitHub issues, pull requests, discussions, documentation, review meetings, demos, and community channels connected to MindMap Care.
