# License Guidance

MindMap Care does not yet have a selected open-source license.

Until a formal license is added, the repository should be treated as shared for review, collaboration, and evaluation only. Do not assume permission for production use, redistribution, commercial use, or derivative works.

## Before Selecting a License

The project owner should decide:

- whether the repository should be open source, source available, or private collaboration only
- whether frontend, backend, documentation, and model artifacts need different licensing terms
- how commercial use should be handled
- whether pilot partners need separate data-use or software-evaluation agreements
- how third-party dependencies and generated assets are licensed

## Common Options to Review

Common software license options include:

- MIT License for permissive open-source use
- Apache License 2.0 for permissive use with explicit patent terms
- GNU AGPLv3 for stronger network-use sharing obligations
- a custom source-available or commercial evaluation license

Documentation may need a separate Creative Commons license if the project owner wants different terms for docs.

## Health-Tech Note

A software license does not replace clinical governance, regulatory review, data protection agreements, validation evidence, or pilot partner contracts.

Add a formal `LICENSE` file before inviting broad external reuse.
